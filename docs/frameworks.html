Frameworks

Frameworks are an example of how PickleDB can be used in common tasks such as user data store, product store, and so on.
Its main class (FrameworkDBScheme) is a modified pickledb class, intended for more flexibility in data storing.
Frameworks have PickleDB built-in, so they all share the same functions.

FrameworkDBScheme Class Functions

search_by by where_is → Searchs for all identifiers that contains 'by' key in its data (available since 0.2)

get_current_id_and_sum → Gets current saved id, and sums one when returned (available since 0.2)


UsersDB Class Functions

add_user → Creates a new user dictionary (available since 0.2)

remove_user → Removes a user dictionary (available since 0.2)


ProductsDB Class Functions

add_product → Creates a new product dictionary (available since 0.2)

remove_product → Removes a product dictionary (available since 0.2)
